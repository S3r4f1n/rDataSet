# rDataSet - Project Context

## Project Overview

**rDataSet** is an R package that treats data frames as mathematical sets. It provides set-theoretic operations (union, intersection, difference, equality) for data frames with a focus on row-level value comparisons using `NA` presence/absence semantics.

### Core Concepts

- **Dataset Structure**: A data frame with designated ID columns (defined via `dataset_build()`)
- **Value Semantics**: Values are identified by `(row_id, col_id)` pairs; presence/absence determined by `is.na()`
- **Set Operations**:
  - `A - B` (difference): Values in A but not in B
  - `A > B` (intersection): Values present in both A and B (taken from A)
  - `A + B` (union): Values in A or B (A takes precedence for overlaps)
  - `A == B` (equality): Cell-wise boolean comparison

### Key Functions

| Function | Description |
|----------|-------------|
| `dataset_build(df, ids)` | Create a dataset from a data frame with specified ID columns |
| `ids(dataset)` | Extract ID columns from a dataset |
| `vals(dataset)` | Extract value columns from a dataset |
| `-.dataset` | Set difference operator |
| `>.dataset` | Set intersection operator |
| `+.dataset` | Set union operator |
| `==.dataset` | Set equality operator |

## Building and Running

### Development Workflow

```r
# Load package for development
devtools::load_all()

# Run tests
devtools::test()
# or
testthat::test()

# Rebuild documentation and NAMESPACE
devtools::document()

# Full package check
devtools::check()
```

### Installation (from source)

```r
install.packages("path/to/rDataSet", repo = NULL, type = "source")
```

### Testing

The package uses testthat (edition 3). Run tests with:

```r
library(testthat)
test_check("rDataSet")
```

## Development Conventions

### Code Style

- Uses roxygen2 comments (`#'`) for documentation
- S3 methods exported with `@export` tag
- Functions use snake_case naming
- Pipeline style with `%>%` (dplyr)

### Testing Practices

- Tests organized by feature area (arithmetics, dataset_building)
- Uses testthat expectations (`expect_equal`, `expect_error`)
- Test fixtures defined at top of test files (e.g., datasets A, B, C)

### Documentation

- Edit `README.Rmd` (not `README.md` directly)
- Use `devtools::build_readme()` to regenerate README.md
- Roxygen comments in `R/sets.R` generate man pages

## Key Implementation Details

### Integrity Checks

- `dataset_integrity()` validates that ID columns uniquely identify rows
- All set operations verify ID column compatibility between operands
- Row/column alignment enforced in equality operations

### Mathematical Properties

The implementation satisfies: `a = (a ∩ b) ∪ (a \ b)`

Tested in `test-arithmetics.R` across all dataset combinations.
